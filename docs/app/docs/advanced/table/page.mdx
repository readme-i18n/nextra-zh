# 表格渲染指南

本文档介绍在MDX中渲染表格的不同方法，包括GFM语法和原生HTML标签。

## GFM语法

在Markdown中，推荐使用[GFM语法](https://github.github.com/gfm/#tables-extension-)编写表格。

```mdx filename="MDX"
| left   | center | right |
| :----- | :----: | ----: |
| foo    |  bar   |   baz |
| banana | apple  |  kiwi |
```

将会被渲染为：

| 左对齐   | 居中对齐 | 右对齐 |
| :----- | :----: | ----: |
| foo    |  bar   |   baz |
| banana | apple  |  kiwi |

## 原生HTML表格

如果尝试使用原生HTML元素`<table>{:js}`、`<thead>{:js}`、`<tbody>{:js}`、`<tr>{:js}`、`<th>{:js}`和`<td>{:js}`来渲染表格，由于[MDX](https://mdxjs.com/docs/using-mdx/#components)不会用`useMDXComponents(){:js}`[^1]提供的组件替换原生HTML元素，表格将无法获得样式。

> [!TIP]
>
> 建议改用[nextra/components](/docs/built-ins/table)提供的[内置`<Table>`组件](/docs/built-ins/table)。

## 修改默认行为

如需使用标准HTML元素但希望应用`useMDXComponents(){:js}`[^1]提供的组件样式，可通过配置Nextra实现。

为此，需向Nextra函数传递`whiteListTagsStyling`选项，包含需要替换的标签数组。

以下是`next.config.mjs`文件中的配置示例：

```js filename="next.config.mjs" {4}
import nextra from 'nextra'

const withNextra = nextra({
  whiteListTagsStyling: ['table', 'thead', 'tbody', 'tr', 'th', 'td']
})

export default withNextra()
```

在此示例中，`<table>`、`<thead>`、`<tbody>`、`<tr>`、`<th>` 和 `<td>` 标签将被替换为对应的 MDX 组件，从而实现自定义样式。

[^1]: https://mdxjs.com/packages/react/#usemdxcomponentscomponents
