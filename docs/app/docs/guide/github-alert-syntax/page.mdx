# GitHub 警示语法

`nextra-theme-docs` 和 `nextra-theme-blog` 支持将
[GitHub 警示语法](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax#alerts)
替换为适用于 `.md`/`.mdx` 文件的 `<Callout>` 组件。

## 使用指南

```md filename="Markdown"
> [!NOTE]
>
> Useful information that users should know, even when skimming content.

> [!TIP]
>
> Helpful advice for doing things better or more easily.

> [!IMPORTANT]
>
> Key information users need to know to achieve their goal.

> [!WARNING]
>
> Urgent info that needs immediate user attention to avoid problems.

> [!CAUTION]
>
> Advises about risks or negative outcomes of certain actions.
```

## 示例

> [!NOTE]
>
> 用户应当了解的有用信息，即使是在快速浏览内容时。

> [!TIP]
>
> 帮助用户更高效或更便捷完成操作的实用建议。

> [!IMPORTANT]
>
> 用户达成目标所需掌握的关键信息。

> [!WARNING]
>
> 需要用户立即关注的紧急信息，以避免出现问题。

> [!CAUTION]
>
> 关于某些操作可能带来风险或负面结果的警示。

## 在自定义主题中使用

若您希望在自定义主题中使用此功能，并搭配自己的 `<Callout>` 组件：

import { Steps } from 'nextra/components'

<Steps>

### 创建 `<Blockquote>` 组件

要创建 `<Blockquote>` 组件，首先需要从 `nextra/components` 导入 `withGitHubAlert`。然后通过调用 `withGitHubAlert` 函数来创建 `<Blockquote>` 组件。

第一个参数应是一个处理 GitHub 提示语法的 React HOC 组件，第二个参数应是标准的 `<blockquote>` 组件。

`type` 属性可以是以下值之一：
`'note' | 'tip' | 'important' | 'warning' | 'caution'{:ts}`。

```jsx
import { withGitHubAlert } from 'nextra/components'

const Blockquote = withGitHubAlert(({ type, ...props }) => {
  return <MyCalloutComponent type={type} {...props} />
}, MyBlockquoteComponent)
```

### 将 `<Blockquote>` 提供给 `useMDXComponents`

要使 `<Blockquote>` 组件可用，应将其集成到 `useMDXComponents` 函数中：

```jsx filename="mdx-components.jsx"
export function useMDXComponents(components) {
  return {
    blockquote: Blockquote,
    ...components
  }
}
```

</Steps>
