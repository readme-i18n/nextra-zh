# Markdown

## MDX

使用 Nextra 时，`content` 目录下的所有 `.mdx` 文件都将通过 [MDX](https://mdxjs.com/about) 渲染，这是一种支持 React 组件的高级 Markdown 格式。

例如，您可以在 MDX 文件中导入并使用 React 组件，如下所示：

```mdx filename="MDX"
## Hello MDX

import { useState } from 'react'

export function Counter({ children }) {
  const [count, setCount] = useState(0)
  return (
    <button onClick={() => setCount(prev => prev + 1)}>
      {children}
      {count}
    </button>
  )
}

<Counter>**Clicks**: </Counter>
```

生成效果：

<Shadow mode="open" className="nextra-border mt-6 rounded-xl border p-4">
  <h2>Hello MDX</h2>
  <Counter>**Clicks**: </Counter>
</Shadow>

除了基础 MDX 功能外，Nextra 还内置了一些高级 Markdown 特性。

## GitHub 风格 Markdown

[GFM](https://github.github.com/gfm) 是 GitHub 创建的 Markdown 扩展，新增了删除线、任务列表、表格等功能支持。

### 删除线

~~已删除内容~~

```md filename="Markdown"
~~removed~~
```

### 任务列表

- [x] 撰写新闻稿
- [ ] 更新网站
- [ ] 联系媒体

```md filename="Markdown"
- [x] Write the press release
- [ ] Update the website
- [ ] Contact the media
```

### 表格

| 语法          |   描述   |       测试文本 |
| :------------ | :------: | -------------: |
| 标题          |   标题   |     示例内容   |
| 段落          |   文本   |       更多内容 |
| 删除线        |          |      ~~文本~~  |

```md filename="Markdown"
| Syntax        | Description |   Test Text |
| :------------ | :---------: | ----------: |
| Header        |    Title    | Here's this |
| Paragraph     |    Text     |    And more |
| Strikethrough |             |    ~~Text~~ |
```

### 自动链接

访问 https://nextjs.org。

```md
Visit https://nextjs.org.
```

### 自定义标题 ID

您可以使用 `## 我的标题 [#自定义ID]` 格式指定自定义标题 ID。例如：

```md filename="Markdown"
## Long heading about Nextra [#about-nextra]
```

在此示例中，`#about-nextra` 将作为标题链接，替代默认生成的 `#long-heading-about-nextra`。
