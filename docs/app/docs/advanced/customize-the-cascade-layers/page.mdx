# 自定义级联层

在某些场景下，你可能需要对 Nextra 预定义的 CSS 进行更精细的控制，以避免级联层中的样式被意外覆盖。以下示例展示了如何通过 [postcss-import](https://github.com/postcss/postcss-import) 将 `nextra-theme-docs` 的预定义 CSS 放置到指定级联层中：

<Steps>

## 安装 `postcss-import`

安装 `postcss-import` 并添加到 `postcss.config.mjs`：

```js filename="postcss.config.mjs"
export default {
  plugins: {
    'postcss-import': {}
    // ... 其他 PostCSS 插件（如 `autoprefixer`、`cssnano`）
  }
}
```

## 配置级联层

在 CSS 文件（如 `styles.css`）中导入 `nextra-docs-theme` 的 CSS 并指定层级：

```css filename="styles.css"
@layer nextra, my-base;

@import 'nextra-theme-docs/dist/style.css' layer(nextra);

@layer my-base {
  /* 基础样式 */
}
```

## 导入 CSS 文件

在应用的顶层布局（如 `app/layout.jsx`）中导入 CSS 文件以应用样式：

```jsx filename="app/layout.jsx"
import '../path/to/your/styles.css'

export default async function RootLayout({ children }) {
  // ... 布局逻辑
}
```

</Steps>
